Instalando o Rust:

Para instalar o rust é preciso acessar o site oficial na parte designada à instalação:
(https://www.rust-lang.org/pt-BR/tools/install)

O método principal utilizado para instalar o rust é com um instalador chamado Rustup. No windows, selecione a versão correspondente ao seu sistema x32 ou x64 bits. Após o download basta instalar que o rust estará pronto para ser utilizado em sua máquina.

Se você estiver utilizando uma distribuição linux será necessário utilizar a biblioteca curl para a instalação do rust, você pode utilizar o seguinte comando para isso:
“sudo apt-get install curl –y”

Se você estiver utilizando a distro RedHat utilize:
“sudo dnf install curl –y”

Agora basta utilizar os próximos comandos para instalar o rust
Para uma distribuição linux utilize no terminal:
“curl https://sh.rustup.rs -sSf | sh”

Caso esteja utilizando o WSL [Windows Subsystem for Linux] utilize o seguinte comando no terminal:
“curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh”

Se você já instalou Rustup no passado, você pode atualizar para uma nova versão executando o comando [rustup update] no terminal do windows, ou pelo powershell.

Criando um projeto – Hello World

Crie um novo diretório com nome de sua preferência. Agora nesta pasta crie um arquivo com a extensão [.rs] e abra no editor de sua preferência.
Para este exemplo estaremos usando [/Rust] como diretório e [main.rs] para o arquivo principal.
Agora neste arquivo escreva o código a seguir e salve:

        
    fn main() { 

    println!("Olá Mundo!"); 
    }

Nota de Sintaxe:

  • fn – Indica uma função ou ponteiro de função
  • println! – printa na tela uma mensagem e segue para a próxima linha
  • print! - equivalente ao println! Exceto que não printa uma nova linha no final da mensagem
Após salvar o arquivo acesse o diretório pelo terminal ou powershell, basta compilar o programa e executar, utilizando:

rustc main.rs
./main

Como resultado para o código você verá na saída do terminal ‘Olá mundo!’

Utilizando Cargo

Cargo é uma ferramenta que permite pacotes Rust declararem suas várias dependências, para isso ele introduz dois arquivos de metadados com vários bits de informações do pacote, Busca e constrói as dependências do seu pacote e chama o rustc para construir seu pacote de forma otimizada.

Para iniciar um novo projeto utilizando cargo utilize o comando [cargo new] + "nome do projeto" no terminal do windows ou pelo powershell, como no exemplo:

cargo new projeto

Como resultado teremos o diretório projeto com os seguintes itens: 

. projeto 

├── Cargo.toml  

└── src  

     └── main.rs

Primeiro vamos analisar cargo.toml, este é um arquivo chamado de [manifesto] e contém todo metadado que o cargo precisa para compilar o seu pacote. 

        
    [package]  

	name = "projeto"  

	version = "0.1.0"  

	authors = ["Your Name "]  

	edition = "2020"  

	[dependencies]

Seguindo para o próximo arquivo main.rs que está dentro da pasta src: 

        
	fn main() {  

	println!("Hello, world!");  

	}

Como podemos observar, a função [cargo new] gera por padrão um arquivo main.rs com a função ‘Hello, world!’ inclusa. Para compilar este cógido gerado com cargo basta acessar o diretório que foi criado pelo terminal e utilizar o comando [cargo build] e para executar basta utilizar [cargo run] o resultado será o "Hello world" no output do terminal assim como descrito na função.

Nesta sequência podemos verificar que foram criadas novas pastas com o arquivo compilado, o caminho gerado foi: [./target/debug/].
Olhando os arquivos verificamos que o nome do executável gerado será idêntico ao [name] indicado em cargo.toml
Quando estiver pronto para lançar uma aplicação você pode usar [cargo build --release] para compilar seus arquivos.

Com [cargo build] e [cargo run] o aplicativo gerado foi compilado em modo de depuração [o padrão para desenvolvimento]. Neste modo o tempo de compilação é menor porque o compilador não faz otimizações, mas o código será executado mais lentamente.
Já utilizando [Cargo build -–release] as otimizações estão ativadas, neste modo de lançamento o programa leva mais tempo para compilar, mas o código será executado mais rápido.

Nota: [cargo build –release] coloca o binário resultante em [target/release] no lugar de [target/debug].

Comentários

Qualquer linguagem precisa de comentários e em rust não é diferente.
Comentários são notas que podem ser incluídas no código fonte para descrever funções, nos ajudar a localizar partes importantes no código e muito mais.

Note que comentários não modificam o programa executado e servem somente para ajudar o programador a melhor organizar seu projeto. 

        

	// comentário em linha. 

	// O comentário se estende até o final desta linha. 

	/* 

	Bloco de comentários 

	Permite comentar em todas as linhas dentro da marcação 

	*/
